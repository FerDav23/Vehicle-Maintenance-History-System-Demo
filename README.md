# Vehicle Maintenance History System – Portfolio Demo

This repository contains a **portfolio demo of a Vehicle Maintenance History System** originally developed for **FJ CIM (Centro Integral de Mantenimiento)**.

The company **granted permission for this project to be shared for portfolio demonstration purposes**.

The original system connects to FJ CIM’s internal maintenance platform, retrieves service records, and presents them in a **clear and user-friendly interface** where customers can review and filter the maintenance history of their vehicles.

To protect **proprietary business logic and internal system architecture**, this public version has been adapted into a **static demo using mock JSON data**.

---

# Project Overview

The goal of this system is to provide customers with **transparent and accessible vehicle maintenance records**.

The interface allows users to explore their vehicle service history in an organized format, improving visibility into past repairs and maintenance events.

Users can:

- View maintenance history for their vehicles
- Filter service records
- Explore vehicle information in a structured interface
- Navigate through service timelines

---

# Responsive Interface

Two interface presentations were developed to improve accessibility across devices.

## Desktop Version
- Optimized for larger screens
- Displays detailed maintenance data
- Designed for easy filtering and exploration of service history

## Mobile Version
- Adapted layout for smaller screens
- Simplified navigation for quick access
- Allows customers to check vehicle records directly from their phones

---

# Technical Challenge

One of the main challenges of this project was **integrating with an already existing system**.

The legacy system had its own architecture and logic, requiring:

- Careful analysis of existing backend workflows
- Understanding how service records and vehicle data were structured
- Adapting new components to work with the current infrastructure
- Designing a new interface while maintaining compatibility with the existing system

This required **reverse engineering portions of the existing system logic and adapting the new interface accordingly**.

---

# Portfolio Version

The original project included **both frontend and backend development** and integrated directly with the company's internal systems.

For security and confidentiality reasons, this public version:

- Uses **mock JSON data instead of real company data**
- Does **not include the original backend**
- Removes any **internal integrations, authentication systems, or proprietary logic**

This ensures that the project can be safely shared while still demonstrating the **interface, architecture, and functionality of the system**.
